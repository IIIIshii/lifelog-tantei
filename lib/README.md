# lib/ — 設計メモ

## ディレクトリ構成

```
lib/
├── core/       # テーマ・カラートークン定義
├── models/     # データ構造の定義
├── pages/      # 画面表示とユーザー操作
├── prompts/    # Geminiへのプロンプト定義
├── services/   # Firebase・Gemini APIとの通信
└── widgets/    # 共通UIパーツ
```

UIとビジネスロジックを層ごとに分離している。

## 主な設計判断

### 1. GeminiServiceのモデルを2つに分けた理由
`lib/services/gemini_service.dart`

インタビュー用（探偵キャラ）と日記生成用でシステムプロンプトの
役割が根本的に異なるため、同一コンテキストに混在させると
出力精度が落ちると判断した。

### 2. 質問フローの順番設計（カスタム→思い出しアシスト→メインの出来事）
`lib/pages/diary_page.dart`

2つの理由がある。
1つ目は離脱リスクの軽減で、負荷の低い質問を先に完了させることで
途中離脱してもデータを確保できる。
2つ目はメインインタビューの質向上で、時系列で記憶をウォームアップ
してからメインの深掘りに入る構造にした。

### 3. フェーズ管理にenumを使った理由
`lib/pages/diary_page.dart`

Stringで管理するとタイポや存在しない値を
コンパイル時に検出できない。
enumにすることで不正な状態への遷移を
コンパイル時に検出できる。

### 4. 質問管理にQueueを使った理由
`lib/pages/diary_page.dart`

質問を先頭から順番に取り出すだけでよいため、
ListのremoveAt(0)がO(n)になるのに対し
QueueのremoveFirst()はO(1)で処理できる。

### 5. サービスクラスへの操作集約（関心の分離）
`lib/services/auth_service.dart` / `lib/services/firestore_service.dart` / `lib/services/gemini_service.dart`

認証・DB操作・AI通信をそれぞれのサービスクラスに集約し、
各ページから実装の詳細を隠蔽した。
将来Firebase以外の認証方式やDBに変更する場合も、
該当サービスクラスだけ修正すれば済む設計にしている。

### 6. 表示用文字列と数値データを分けて保存した理由
`lib/services/firestore_service.dart`

「7時間」という文字列は画面表示に使い、
7.0という数値は将来の証拠分析室（グラフ）での集計に使う。
未実装機能のために先回りして設計している。

### 7. 一覧画面にStreamを使った理由
`lib/pages/diary_list_page.dart`

Futureは1回だけ取得するのに対し、Streamはデータ更新を
リアルタイムで受け取り続けるため、
手動リロードなしに一覧を更新できる。

### 8. UserSettingsをイミュータブルにした理由
`lib/models/user_settings.dart`

finalフィールドのみで構成し、変更時はcopyWithで
新しいインスタンスを生成する設計にした。
状態の意図しない書き換えを防ぐためである。

### 9. AIプロンプトを専用ファイルに分離した理由
`lib/prompts/ai_instructions.dart` / `lib/prompts/diary_prompts.dart`

探偵のキャラクター定義や日記フォーマットの指示を
専用ファイルに一元管理することで、
口調やフォーマットを変更する際に
該当ファイルだけ修正すれば済む設計にしている。

### 10. UID取得完了までボタンを無効化した理由
`lib/pages/home_page.dart`

非同期で取得するUIDが確定する前に画面遷移が発生すると
Firestore操作でクラッシュするため、
UID取得完了まで操作を無効化している。
VoidCallback?のnull許容を利用して実装した。

## 既知の改善点

### DONE判定の脆弱性
`lib/services/gemini_service.dart`

現状、AIインタビューの終了判定が `'DONE'` の完全一致のみ。
AIの出力は非決定的なため、`'DONE。'` や `'done'` が
返った場合に終了判定が機能しない可能性がある。

```dart
// 現状
if (text == 'DONE') return null;

// 改善案
if (text.trim().toUpperCase().contains('DONE')) return null;
```

（Issue管理済み）

## Dartの文法メモ（本プロジェクトで学んだもの）

### 変数・型

| 記法 | 意味 |
|------|------|
| `String?` | null許容型。値がnullかもしれない |
| `final` | 一度代入したら変更できない変数 |
| `const` | コンパイル時に値が確定する定数 |
| `dynamic` | どんな型でもOKな型 |
| `_変数名` | privateの宣言。ライブラリ外（通常はファイル外）から触れない |

### null安全

| 記法 | 意味 |
|------|------|
| `??` | null合体演算子。nullなら右側の値を使う |
| `!` | null断言演算子。「絶対nullじゃない」と宣言 |
| `?.` | null安全アクセス。nullなら処理をスキップ |

### 関数・クラス

| 記法 | 意味 |
|------|------|
| `=>` | アロー関数。1行で返せる時の省略記法 |
| `(x) => x * 2` | アロー関数の引数あり版 |
| `{引数}` | 名前付き引数（省略可能）。呼び出し時に名前を指定する |
| `{引数 = デフォルト値}` | 省略時のデフォルト値を持つ名前付き引数 |
| `extends` | 親クラスの機能を引き継ぐ（継承） |
| `@override` | 親クラスのメソッドを上書きする |
| `super.key` | 親クラスのkeyをそのまま渡す省略記法 |
| `this.フィールド名` | 自分自身のフィールドを参照する |
| `factory` | インスタンスの生成方法を自由に定義できるコンストラクタ |
| `static` | インスタンス化せずに呼べるメソッド・変数 |
| `クラス名._()` | privateコンストラクタ。外からインスタンス化できないようにする |
| `: フィールド = 値` | イニシャライザリスト。finalフィールドをコンストラクタ本体より先に初期化する |

### 非同期

| 記法 | 意味 |
|------|------|
| `Future<T>` | 将来Tを返すことを約束する型 |
| `Future<void>` | 将来何も返さない（処理するだけ） |
| `async` | 「この関数の中でawaitを使う」という宣言 |
| `await` | 非同期処理の結果が来るまで待つ |

### コレクション

| 記法 | 意味 |
|------|------|
| `List<T>` | 順序付きリスト |
| `Map<K, V>` | キーと値のペアの辞書型 |
| `<String, dynamic>{}` | 空のMapを作る省略記法 |
| `Queue<T>` | 先入れ先出し（FIFO）のデータ構造。先頭取り出しがO(1) |
| `.map((x) => ...)` | リストの各要素を変換する |
| `.where((x) => ...)` | 条件に合う要素だけを取り出す（SQLのWHEREと同じ） |
| `.join('区切り文字')` | リストを1つの文字列に結合する |
| `.removeFirst()` | Queueの先頭要素を取り出して削除する |

### Flutter固有

| 記法 | 意味 |
|------|------|
| `StatelessWidget` | 変化しないWidget |
| `StatefulWidget` | 状態が変化するWidget |
| `setState(() {...})` | 状態を変更して画面を再描画する |
| `ValueListenableBuilder` | 監視対象の値が変わったら自動で再描画する仕組み |
| `VoidCallback?` | 返り値のない関数を受け取るnull許容の型 |
| `Stream<T>` | データが流れ続けるリアルタイムな値の流れ |
| `StreamBuilder` | Streamの更新を受け取って自動で画面を再描画するWidget |

### その他

| 記法 | 意味 |
|------|------|
| `enum` | 列挙型。決まった値の集合を定義する |
| `switch (_phase) { case ...: }` | 列挙型の値に応じて処理を分岐する |
| `条件 ? A : B` | 三項演算子。条件がtrueならA、falseならB |
| `as 型名` | 型キャスト。「この値をこの型として扱う」 |
| `$変数名` | 文字列への変数埋め込み（文字列補間） |
| `\n` | 改行文字 |
| `..sort(...)` | カスケード記法。同じオブジェクトに続けて操作する |
| `_` | 使わない引数の慣習的な名前 |
| `import 'package:xxx/xxx.dart'` | 外部パッケージの読み込み |
| `import '../xxx/xxx.dart'` | プロジェクト内ファイルの読み込み |
